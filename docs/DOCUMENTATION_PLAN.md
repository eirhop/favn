# Favn Documentation Plan

> **Historical:** The documentation-tree migration described by this plan is
> complete. This file remains only to explain the resulting layout; it is not an
> active backlog or required agent reading.

The migration established these sources of truth:

- `apps/favn/README.md` and `apps/favn/guides/` for public HexDocs content;
- public moduledocs and typespecs for API reference;
- `docs/structure/` for application ownership;
- `docs/architecture/` and `docs/storage/` for internal technical contracts;
- `docs/operators/` and `docs/production/` for operational contracts;
- `docs/FEATURES.md` for shipped capability and `docs/ROADMAP.md` for future work;
- `docs/report/`, `docs/refactor/`, and `docs/archive/` for historical evidence.

Current navigation is [`README.md`](README.md). Documentation changes follow
[`DOCUMENTATION_GUIDE.md`](DOCUMENTATION_GUIDE.md). New work should update those
canonical locations and GitHub issues rather than extending this plan.

# Runtime Error Handling and Monitoring Review

Priority: P1 - production operability.

Reviewed after fast-forwarding the worktree to `origin/main` at `ada8d15`.
Scope covered the requested structure docs and static inspection of:

- `apps/favn_orchestrator`
- `apps/favn_runner`
- `apps/favn_view`
- `apps/favn_sql_runtime`
- `apps/favn_duckdb`
- `apps/favn_duckdb_adbc`
- `apps/favn_storage_postgres`
- `apps/favn_storage_sqlite`

## Top Production Observability Gaps

1. Orchestrator has custom operational logs and an optional metrics hook, but no standard `:telemetry` events for run transitions, scheduler ticks/submissions, admission waits, persistence failures, cancellation dispatch, or projection failures.
2. Several control-plane failure paths can leave runner work active while persisted orchestrator state says failed, cancelled, or is otherwise uncertain.
3. Runner and SQL runtime do not have bounded execution, checkout, query, materialization, transaction, or native/DB cancellation contracts for long-running work.
4. Storage and SQL adapters return raw low-level errors in multiple paths and lack consistent operation telemetry by adapter, operation, table/relation, and error class.
5. Readiness is shallow in important paths; Postgres storage can be reported ready because the facade defaults adapters without readiness callbacks to healthy.

## Missing Monitoring And Telemetry

| Finding | Location | Missing Signal | Recommended Contract |
| --- | --- | --- | --- |
| Orchestrator operational events are logs plus a custom hook, not standard telemetry. | `apps/favn_orchestrator/lib/favn_orchestrator/operational_events.ex:21-35`, `emit/4` | Dashboards cannot subscribe to standard `:telemetry`; log metadata is stringified. | Emit `:telemetry.execute/3` for `[:favn, :orchestrator, :run, :transition]`, scheduler, admission, storage, and projection events. Use Logger metadata rather than interpolated strings. |
| Admission queues lack queue-depth, wait-age, timeout, and wakeup-effectiveness metrics. | `apps/favn_orchestrator/lib/favn_orchestrator/execution_admission.ex:34-52`, `acquire_or_wait/3`; `apps/favn_orchestrator/lib/favn_orchestrator/execution_admission/coordinator.ex:61-69`, `handle_cast({:notify_scopes, ...})` | No production visibility into capacity saturation or stale waiters. | Emit `admission_queued`, `admission_acquired`, `admission_wait_expired`, and `admission_wakeup_sent` with `run_id`, `asset_step_id`, `queue_reason`, `scope_kind`, `scope_key`, `wait_ms`, and `queue_depth`. |
| Scheduler readiness only checks process existence. | `apps/favn_orchestrator/lib/favn_orchestrator/readiness.ex:61-72`, `scheduler_check/0`; diagnostics exist at `apps/favn_orchestrator/lib/favn_orchestrator/scheduler/runtime.ex:125-140` | Scheduler can be alive but lagged, unable to persist, or stuck. | Read scheduler diagnostics in readiness and expose `last_tick_at`, `tick_lag_ms`, `queued_schedule_count`, `in_flight_schedule_count`, and `last_error`. |
| Scheduler state writes can crash the scheduler with no telemetry. | `apps/favn_orchestrator/lib/favn_orchestrator/scheduler/runtime.ex:279-282`, `load_states/1`; `apps/favn_orchestrator/lib/favn_orchestrator/scheduler/runtime.ex:319-323`, `evaluate_all/1` | `:ok = Storage.put_scheduler_state(...)` hides persistence failure until crash/restart. | Handle `{:error, reason}` explicitly, degrade scheduler, and emit `[:favn, :orchestrator, :scheduler, :state_persist_failed]`. |
| Runner execution lifecycle lacks telemetry for start, stop, exception, cancel, and await-timeout. | `apps/favn_runner/lib/favn_runner/server.ex:130-172`, `submit_work/2`; `apps/favn_runner/lib/favn_runner/server.ex:289-323`, result/crash handlers; `apps/favn_runner/lib/favn_runner/server.ex:365-377`, `finalize_execution/3` | Execution failures are mostly state transitions and runner logs, not production metrics. | Emit `[:favn, :runner, :execution, :start | :stop | :exception | :cancel | :await_timeout]` with `execution_id`, `run_id`, `asset_ref`, `status`, `duration_ms`, `error_type`, and `retryable?`. |

| Storage adapter DB calls lack operation telemetry and normalized error contracts. | `apps/favn_storage_postgres/lib/favn/storage/adapter/postgres.ex:1556-1573`, `query_and_decode_rows/4`; `apps/favn_storage_postgres/lib/favn/storage/adapter/postgres.ex:1600-1604`, `query_ok/3`; `apps/favn_storage_sqlite/lib/favn/storage/adapter/sqlite.ex:2176-2180`, `query_ok/3`; `apps/favn_storage_sqlite/lib/favn/storage/adapter/sqlite.ex:2247-2251`, `query_distinct_backfill_ids/3` | No latency or error-rate signal by storage operation/table; raw `%Postgrex.Error{}` and `%Exqlite.Error{}` can propagate. | Normalize storage errors to `%FavnOrchestrator.Storage.Error{type, operation, adapter, retryable?, classification}`. | Postgres storage readiness is missing, while the storage facade defaults unsupported adapters to ready. | `apps/favn_orchestrator/lib/favn_orchestrator/storage.ex:44-51`, `readiness/0`; `apps/favn_storage_postgres/lib/favn/storage/adapter/postgres.ex:1-90`, module lacks `readiness/1` and `diagnostics/1` | External repo/schema/connectivity failures can be reported healthy. | Implement `readiness/1` and `diagnostics/1` for Postgres with connectivity, migration/schema readiness, repo mode, and redacted failure reason. |

## Unclear Ownership And Boundary Violations

| Finding | Location | Risk | Recommended Contract |
| --- | --- | --- | --- |
| View renders raw backend error terms with `inspect(reason)`. | `apps/favn_view/lib/favn_view/pipelines_live.ex:63-67`; `apps/favn_view/lib/favn_view/runs_list_live.ex:167-171`; `apps/favn_view/lib/favn_view/runs_list_live.ex:288-294`; `apps/favn_view/lib/favn_view/pipeline_detail_live.ex:403-405`; `apps/favn_view/lib/favn_view/run_detail_live.ex:866-875` | Storage/plugin/orchestrator internal errors can leak to browser and operator screenshots. | Public orchestrator facade should return stable user-facing error atoms/structs. View should show generic labels and log the raw redacted reason with a correlation ID. |
| DuckDB transaction errors include formatted stack traces and raw cause in returned `%Favn.SQL.Error{}`. | `apps/favn_duckdb/lib/favn/sql/adapter/duckdb.ex:1196-1214`, transaction rescue; `apps/favn_duckdb_adbc/lib/favn/sql/adapter/duckdb/adbc.ex:1214-1231` | Low-level exceptions and possibly secrets can cross adapter boundaries into runner/view/API paths. | Return redacted SQL errors with `exception_module`, `transaction_stage`, and `classification`; log full stack server-side with correlation ID. |
| SQL client passes raw adapter connect/bootstrap/operation errors through in some paths. | `apps/favn_sql_runtime/lib/favn/sql/client.ex:750-765`, `connect_and_build_session/4`; `apps/favn_sql_runtime/lib/favn/sql/client.ex:860-873`, `run_owned_session_operation/5` | Boundary contracts vary by adapter; pooled sessions may not be discarded for non-`%Favn.SQL.Error{}` failures. | Normalize every adapter error into `%Favn.SQL.Error{operation, adapter, connection, retryable?, classification}` and mark pooled sessions for discard on any non-read operation error unless explicitly pool-safe. |
